Artifact Data Entry
Enter the appropriate information from the accession records into each field on the editor document. The only field that may never be duplicated from another artifact is the accession number.

Date Entry Syntax
Dates must be entered with appropriate syntax to specify formatting and exact time. The syntax layout is as follows: 
  formatString|dateString

Optionally, a literal may be specified at the end using the '=' character: 
  formatString|dateString=literalString

The literal will be the text displayed on the ArtifactView, while the date string will still be parsed and processed as the attributed time.

The format string is comprised of letters to represent how you are going to format the actual date (which comes after the '|' pipe symbol). For example: 
  MM/dd/yyyy would specify format for a time value such as: 03/08/1905

N.B. Make sure to include leading zeros for double digits (i.e 01)!

The general standard format for most dates, if possible, should be: 
  MMM dd yyyy

This would then be entered as such: 
  MMM dd yyyy|Oct 31 2010

You can also specify a time range using dashes: 
  yyyy-yyyy|1930-1980

This will make the time attribute be specified by that entire period of time instead of one specific date. You could then speicfy a literal that says "20th century" or "c.1960" if desired.

N.B. If you only specify a year, the assumed exact date will be January 01. If you specify a month but no date, the assumed date is the first day of the month.
